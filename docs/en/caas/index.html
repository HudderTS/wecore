What CaaS is, how it works, and whether it fits your workload

CaaS delivers a fully deployed OpenStack private cloud on dedicated bare metal servers, managed by WECORE. You get a 3-node (or larger) cluster with OpenStack control plane services, hyper-converged Ceph storage, and private networking — all provisioned in minutes. You get root access to the control plane; we handle hardware, platform upgrades, and 24/7 monitoring.

A standard 3-node OpenStack + Ceph cluster deploys in under 10 minutes. Larger clusters scale linearly — a 5-node deployment completes in under 15 minutes. You receive API credentials and Horizon dashboard access the moment provisioning finishes.

Yes. You get SSH access and admin credentials to the OpenStack control plane. Tune Nova scheduler policies, modify Neutron networking, adjust Ceph placement groups — the stack is yours. We manage the hardware (drive replacement, firmware, kernel patching) and orchestrate OpenStack version upgrades with no tenant downtime.

Cloud Servers and HPCaaS are hourly, pay-per-use — you pay for the resources you consume. CaaS is fixed monthly pricing per cluster: one predictable bill regardless of how many VMs you run or how much data you transfer internally. This makes it ideal for steady-state workloads, SaaS platforms, and compliance-heavy environments.

Yes. You can attach NVIDIA GPU nodes to your cluster for ML training, inference, transcoding, or visualization workloads. GPU nodes are billed as an add-on to your monthly cluster rate.

WECORE manages: hardware lifecycle, OpenStack control-plane version upgrades, Ceph cluster health and rebalancing, network fabric, 24/7 monitoring and incident response. You manage: VMs, tenant networks, security groups, images, volumes, and your applications. Think of it as a managed platform — we run the cloud so you can run your workloads.

Ceph is deployed as a hyper-converged distributed storage system across all nodes in your cluster. Data is replicated 3× for durability. Ceph provides three interfaces: RBD (block storage for VM disks, integrated with Cinder), RADOSGW (S3-compatible object storage), and CephFS (shared POSIX filesystem). No external SAN or NAS appliance is needed.

Yes. Many teams migrate from VMware to avoid Broadcom licensing costs. OpenStack provides equivalent (or better) capabilities: Nova replaces vCenter compute management, Neutron replaces NSX networking, Cinder + Ceph replace vSAN storage, and Keystone replaces vSphere RBAC. We provide migration guidance and can help size a replacement cluster.
