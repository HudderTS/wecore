How it works

From catalog to results, without the cluster babysitting.

Every app follows the same governed path on your WECORE HPC — the same flow the Cloud-Panel marketplace uses for OpenFOAM today.
01

Pick an app

Choose a curated build from the catalog. Defaults for solver, MPI layout, and runtime size are pre-filled so you start from a working baseline.

02

Configure the run

Set inputs, node flavor, core counts, and a stop policy. A cost estimate is shown before anything queues, so there are no surprises.

03

Submit to Slurm

Dispatch to Slurm batch jobs, arrays, or an interactive remote-desktop session through Open OnDemand — all on resources you own.

04

Track reproducible results

Status, logs, convergence, outputs, and accounting stay attached to one durable simulation record you can revisit or clone.
