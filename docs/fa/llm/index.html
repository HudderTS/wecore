جریان کار

از مدل تا endpoint، بدون خروج از ابر شما.

با الهام از راه‌اندازِ Azimuth LLM، وی‌کُر یک مدل متن‌باز انتخاب‌شده را روی نودهای GPU خودتان به یک اپ گفت‌وگو و API در حال اجرا تبدیل می‌کند — بدون نیاز به ساختِ زیرساخت استنتاج.
۰۱

یک مدل انتخاب کنید

از میان مدل‌های متن‌باز منتخب انتخاب کنید یا به هر مخزن پشتیبانی‌شده توسط vLLM روی Hugging Face اشاره کنید — instruct، چت، کد، embedding یا زبانی‑تصویری.

۰۲

اندازهٔ GPU را تعیین کنید

یک فلیور GPU و تعداد replica را از استخرهای Kubernetes وی‌کُر خود انتخاب کنید. بررسی‌های پیش‌پرواز پیش از اجرا تأیید می‌کنند که VRAM و سهمیه با مدل می‌خوانند.

۰۳

در یک گام راه‌اندازی کنید

وی‌کُر vLLM و یک اپ وب گفت‌وگو را برای شما مستقر می‌کند — وزن‌های مدل دریافت، سرویس‌دهی و سلامت‌سنجی می‌شوند و پشت ورود یکپارچه (SSO) عرضه می‌گردند.

۰۴

گفت‌وگو و یکپارچه‌سازی کنید

از گفت‌وگوی وب داخلی استفاده کنید یا اپ‌هایتان را به endpoint سازگار با OpenAI وصل کنید. توکن بر ثانیه، تأخیر و مصرف GPU را در Grafana ببینید.

endpoint سازگار با OpenAI

مقیاس‌گذاری خودکار روی Kubernetes

replicaها به‌صورت یک سرویس روی استخرهای GPU روی KaaS وی‌کُر اجرا می‌شوند، با سلامت‌سنجی، به‌روزرسانی چرخشی و گسترش زیر بار.

رصدپذیری داخلی

یک ServiceMonitor و داشبوردهای آمادهٔ Grafana از همان روز اول توان عملیاتی، تأخیر، عمق صف و بهره‌وری GPU را نمایان می‌کنند.

خصوصی به‌صورت ذاتی

پرامپت‌ها، پاسخ‌ها و وزن‌ها روی فضای ذخیره‌سازی اپراتور درون ابر شما می‌مانند — بدون فراخوانی API شخص ثالث و بدون خروج داده به فروشندهٔ مدل.

پیش‌بینی سهمیه و هزینه

پیش از راه‌اندازی یک مدل، VRAM، ساعت‌های GPU و اثر کیف‌پول را برآورد کنید و لحظه‌ای که replicaها را به صفر می‌رسانید دیگر هزینه‌ای ندهید.

پشتهٔ متن‌باز

بهترین استنتاج متن‌باز، گردآوری‌شده و حاکمیت‌شده.

LLMaaS همان اجزایی را که تیم‌های پژوهش و پلتفرم پیش‌تر به آن اعتماد دارند، پشت یک جریان راه‌اندازی خودگردان بسته‌بندی می‌کند.

پرسش‌های متداول

پرسش‌های سرویس مدل زبانی، پاسخ داده شد

راه‌انداز چیست، چه چیزی سرویس‌دهی می‌کند و چطور با زیرساخت وی‌کُر جور می‌شود

این یک راه‌انداز LLM خوداِمیزبان است: شما یک مدل متن‌باز انتخاب می‌کنید و وی‌کُر آن را روی نودهای GPU شما به‌همراه یک رابط گفت‌وگوی وب و یک API سازگار با OpenAI مستقر می‌کند. این الگو از اپ‌های Azimuth LLM شرکت StackHPC الهام گرفته و برای اجرا روی زیرساخت Kubernetes، GPU و OpenStack وی‌کُر سازگار شده است.

هر مدلی که vLLM پشتیبانی می‌کند و از Hugging Face سرویس‌دهی می‌شود — از جمله Llama، Qwen، Mistral، DeepSeek، Gemma و مدل‌های زبانی‑تصویری. ما پیش‌تنظیم‌های منتخب با پیش‌فرض‌های منطقی عرضه می‌کنیم و شما می‌توانید به مخزن خودتان اشاره کنید؛ وزن‌های دارای دروازه با توکن دسترسی خودتان دریافت می‌شوند.

از طریق یک API گفت‌وگو و تکمیلِ /v1 سازگار با OpenAI. SDKها، چارچوب‌های عامل و ابزارهای موجود تنها با تغییر base URL کار می‌کنند — بدون بازنویسی. یک گفت‌وگوی وب داخلی (و یک رابط تحلیل تصویر برای مدل‌های بینایی) هم برای کسانی که نمی‌خواهند کد بنویسند گنجانده شده است.

هیچ‌جای بیرون از ابر شما. پرامپت‌ها، پاسخ‌ها، وزن‌های مدل و لاگ‌ها روی فضای ذخیره‌سازی اپراتور درون زیرساخت وی‌کُر شما می‌مانند. هیچ فراخوانی API شخص ثالث و هیچ خروج داده‌ای به فروشندهٔ مدل وجود ندارد، که آن را برای بارهای کاری حساس و تحت‌مقررات بسیار مناسب می‌کند.

LLMaaS به‌صورت یک سرویس روی استخرهای نود GPU Kubernetes وی‌کُر (KaaS) شما اجرا می‌شود. هر مدل یک deployment با سلامت‌سنجی، به‌روزرسانی چرخشی و گسترش زیر بار است؛ دسترسی از طریق ingress و پشت ورود یکپارچه (SSO) عرضه می‌شود.

بله. یک ServiceMonitor و داشبوردهای آمادهٔ Grafana توکن بر ثانیه، تأخیر، عمق صف و بهره‌وری GPU را نمایان می‌کنند. چون روی GPUهایی اجرا می‌کنید که از پیش بابتشان هزینه می‌دهید، هزینه همان زمان GPU است — و می‌توانید هنگام بیکاری مدل، replicaها را به صفر برسانید.

KaaS، Kubernetes مدیریت‌شده و استخرهای GPU را در اختیار شما می‌گذارد؛ Cloud HPC، محاسبهٔ دسته‌ای مبتنی بر Slurm را. LLMaaS لایهٔ محصول روی این‌هاست — این بنیان‌ها را به یک راه‌انداز یک‌گامی برای سرویس‌دهی مدل‌های متن‌باز با رابط گفت‌وگو و API تبدیل می‌کند، تا خودتان پشتهٔ استنتاج را سرِهم نکنید.

به‌زودی

GPUهای خود را به یک endpoint خصوصی هوش مصنوعی تبدیل کنید.

LLMaaS وی‌کُر برای تیم‌هایی است که راحتیِ یک API میزبانی‌شدهٔ LLM را می‌خواهند بدون فرستادن پرامپت‌ها و داده به یک فروشندهٔ عمومی — مدل‌های متن‌باز، یک endpoint سازگار با OpenAI و یک رابط گفت‌وگو، همه روی زیرساختی که خودتان کنترل می‌کنید.
